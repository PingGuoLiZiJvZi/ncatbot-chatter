## 2. 最终数据流（实现总图）

```text
ncatbot callback
  ↓ (唯一切入点: put raw_event)
IncomingMessageQueue.put(raw_event)
  ↓
MainLoop 每轮开始:
  MessageIngestor.drain(max_items=50)
  ├─ 解析 → Message
  ├─ 去重 (raw_message_log UNIQUE message_id)
  └─ 写入 MemoryManager.ShortTermMemory
  ↓
Orchestrator:
  1. drain StateEventQueue → BotState.apply()
  2. BotState.tick(now=datetime.now())     ← 基于真实 dt
  3. RuntimeMode 分支检查
  4. DecisionEngine.decide(tick_type)      ← 纯决策
  5. action_log.record_planned(plans)
  6. ContentGenerator.generate(plans)       ← DEGRADED 时走模板
  7. action_log.record_generated(generated)
  8. EmergencyBrake.final_check(generated)
  9. DelaySendScheduler.schedule(valid)
 10. action_log.record_scheduled(valid)
  ↓
DelaySendScheduler 线程:
  到期:
    1. pre_send_check(action, memory)      ← 纯规则复查
    2. action_log.mark_as_sending(id)       ← 原子幂等
    3. BotAdapter.send()
    4. action_log.record_send_result()
    5. StateEventQueue.put(event)
  ↓
下一轮 MainLoop:
  BotState.apply(StateEvent)
```

## 3. 线程模型（最终版）

```
┌─────────────┐
│ ncatbot 线程 │  ← 只做一件事: IncomingMessageQueue.put(raw_event)
│ (事件回调)    │     不解析、不去重、不写 MemoryManager
└──────┬──────┘
       │
       ▼
┌──────────────────┐
│IncomingMsgQueue  │  ← 线程安全 queue.Queue
└────────┬─────────┘
         │
         ▼
┌────────────────────────────────────────────────────────┐
│              主循环线程 (唯一写入 BotState)               │
│                                                        │
│  drain IncomingMessageQueue → MessageIngestor          │
│  drain StateEventQueue     → BotState.apply()          │
│  BotState.tick(now)        → 基于 dt 更新情绪/能量       │
│  passive_tick  → Orchestrator                          │
│  active_tick   → Orchestrator                          │
│  concentrate_tick → Orchestrator                       │
│                                                        │
│  独家写入: BotState, ShortTermMemory._read/_pending     │
│  独家调度: ContentGenerator, ConcentrateJob            │
└────────┬───────────────────────────────────────────────┘
         │ schedule(GeneratedAction + delay)
         ▼
┌────────────────────────────────────────────────────────┐
│            DelaySendScheduler 线程 (唯一发送线程)         │
│                                                        │
│  优先队列 + 单线程消费                                   │
│  pre_send_check (纯规则)                                │
│  BotAdapter.send()                                     │
│  action_log 写入                                        │
│  StateEventQueue.put(event)  ← 通知主循环，不直写状态     │
│                                                        │
│  不写入: BotState, ShortTermMemory                     │
└────────────────────────────────────────────────────────┘
```

**硬规则：**
- BotState 写入：仅主循环线程
- ShortTermMemory._read/_pending 写入：仅主循环线程
- ShortTermMemory._unread 写入：通过 `add_incoming()` with lock；消费通过 `consume_unread()` with lock
- 发送：仅 DelaySendScheduler 线程
- 其他线程不直接跨线程写任何共享状态

### 6.3 `MessageIngestor`（最终版：主循环内执行）

```python
class MessageIngestor:
    """
    由主循环线程在每轮开始时调用 drain()。
    不在 ncatbot 回调线程中执行，避免阻塞 ncatbot 事件循环。
    """
    
    def __init__(self, queue: queue.Queue, memory: MemoryManager, raw_log: RawMessageLog):
        self._queue = queue
        self.memory = memory
        self.raw_log = raw_log
    
    def drain(self, max_items: int = 50) -> int:
        """非阻塞消费一轮，返回处理的消息数"""
        count = 0
        for _ in range(max_items):
            try:
                raw_event = self._queue.get_nowait()
            except queue.Empty:
                break
            msg = self._parse(raw_event)
            if msg and self.raw_log.insert_if_new(msg):
                self.memory.add_message(msg)
                count += 1
        return count

class RawMessageLog:
    """去重表 + 定期清理"""
    
    def insert_if_new(self, msg: Message) -> bool:
        """INSERT OR IGNORE，返回是否为新消息"""
    
    def cleanup(self, ttl_days: int) -> int:
        """删除 received_at < now - ttl_days 天 的记录，返回删除数"""
```

## 7. 记忆层设计

### 7.1 `ShortTermMemory`（原子消费）

```python
class ShortTermMemory:
    _unread: list[Message]       # MessageIngestor 写入 (with lock)
    _read: list[Message]         # 主循环线程独占写入
    _pending: list[Message]      # 主循环线程独占写入
    _self_sent: list[SelfMessage]
    _lock: threading.Lock
    
    def add_incoming(self, msg: Message):
        """线程安全：MessageIngestor 调用"""
        with self._lock:
            self._unread.append(msg)
    
    def consume_unread(self) -> list[Message]:
        """线程安全：主循环调用。原子交换，取走全部 unread。"""
        with self._lock:
            unread = self._unread
            self._unread = []
        self._read.extend(unread)  # 无锁，仅主循环线程执行
        return unread
```

## 8. 状态层设计

### 8.1 `BotState`（基于真实 dt 的时间驱动）

```python
class BotState:
    """
    所有状态更新基于真实时间差 dt_seconds，不依赖 tick 次数。
    由主循环线程独占写入。
    """
    mode: RuntimeMode = RuntimeMode.STOPPED
    last_tick_at: datetime | None = None
    
    valence: float = 0.0          # -1.0 ~ 1.0
    energy: float = 1.0           # 0.0 ~ 1.0
    social_battery: float = 1.0   # 0.0 ~ 1.0
    
    current_hour: int = 0
    activity_weight: float = 0.0
    per_chat: dict[str, ChatState] = {}
    cycle_count: int = 0
    
    _consecutive_llm_failures: int = 0
    
    def tick(self, now: datetime):
        """时间驱动的状态更新。由 Orchestrator 每轮开始时统一调用。"""
        if self.last_tick_at is None:
            self.last_tick_at = now
            return
        
        dt_seconds = (now - self.last_tick_at).total_seconds()
        if dt_seconds <= 0:
            return
        
        # 所有衰减/恢复基于 dt_seconds
        self._update_valence(dt_seconds)
        self._update_energy(dt_seconds)
        self._update_social_battery(dt_seconds)
        
        self.current_hour = now.hour
        self.activity_weight = self.activity_curve.get_weight(now.hour)
        self.last_tick_at = now
    
    def _update_energy(self, dt_seconds: float):
        dt_hours = dt_seconds / 3600
        is_sleeping = self.activity_curve.is_sleeping()
        regen_rate = self.config.emotion.energy_regen_per_hour
        if is_sleeping:
            regen_rate *= self.config.emotion.sleep_regen_multiplier
        self.energy = min(1.0, self.energy + regen_rate * dt_hours)
    
    def _update_valence(self, dt_seconds: float):
        # 向 0 回归，但不越过 0
        regression = self.config.emotion.valence_regression * (dt_seconds / 60)
        if self.valence > 0:
            self.valence = max(0.0, self.valence - regression)
        elif self.valence < 0:
            self.valence = min(0.0, self.valence + regression)
    
    def apply(self, event: StateEvent):
        """在主循环线程内应用状态变更（见 StateEvent 定义）"""
```

### 8.2 `RuntimeMode` 行为表（Orchestrator 实现依据）

| RuntimeMode  | ingest | passive_tick | active_tick | concentrate | 内容生成 | LLM 决策 |
|-------------|--------|-------------|------------|------------|---------|---------|
| RUNNING     | 是     | 是          | 是         | 是         | LLM     | LLM     |
| PASSIVE_ONLY| 是     | 是          | 否         | 否         | LLM     | 否      |
| PAUSED      | 是*    | 否          | 否         | 否         | 无      | 否      |
| DEGRADED    | 是     | 是          | 否         | 否         | 模板    | 否      |
| ERROR       | 是     | 否          | 否         | 否         | 无      | 否      |
| STOPPING    | 一次\* | 否          | 否         | 否         | 无      | 否      |
| STOPPED     | 否     | 否          | 否         | 否         | 无      | 否      |
| STARTING    | 否     | 否          | 否         | 否         | 无      | 否      |

> \* PAUSED: ingest 继续（保留消息），但不回复。
> \* STOPPING: 仅一次性 drain 已入队消息（消费队列中已接收但未处理的消息），之后不再接收新消息。

```python
class Orchestrator:
    def run_passive_tick(self):
        self._drain_ingest()
        self._drain_state_events()
        self.state.tick(datetime.now())

        # RuntimeMode 分支
        if self.state.mode in (RuntimeMode.PAUSED, RuntimeMode.ERROR,
                                RuntimeMode.STOPPING, RuntimeMode.STOPPED,
                                RuntimeMode.STARTING):
            return

        # DEGRADED 健康检查: 在 decision 之前执行，恢复后本轮即可走 LLM 路径
        if self.state.mode == RuntimeMode.DEGRADED:
            if self.state.cycle_count % 10 == 0:
                try:
                    self.llm_client.health_check()
                    self.state_events.put(StateEvent(
                        event_id=uuid4(), event_type=StateEventType.LLM_RECOVERED,
                        payload={"tick_type": "passive_health_check"}, timestamp=datetime.now()
                    ))
                    # apply() 会在下一轮 drain_state_events 时切回 RUNNING
                    # 本轮 mode 仍为 DEGRADED，继续走模板路径
                except LLMError:
                    pass  # 仍然不可用，继续 DEGRADED

        decision = self.engine.decide(TickType.PASSIVE)
        if not decision.plans:
            return

        self.action_log.record_planned(decision.plans)

        if self.state.mode == RuntimeMode.DEGRADED:
            generated = self.degraded_policy.generate(decision.plans)
        else:
            try:
                generated = self.content_gen.generate(decision.plans)
            except LLMError as e:
                self.state_events.put(StateEvent(
                    event_id=uuid4(), event_type=StateEventType.LLM_FAILED,
                    payload={"tick_type": "passive", "chat_ids": [p.chat_id for p in decision.plans]},
                    timestamp=datetime.now()
                ))
                # LLM 失败：已 planned 的 action 标记为 cancelled，避免僵尸记录
                self.action_log.record_batch(decision.plans, status="cancelled", reason="llm_failed")
                generated = []
        
        self.action_log.record_generated(generated)
        
        to_send = []
        for g in generated:
            decision = self.brake.final_check(g)
            match decision:
                case BrakeDecision.ALLOW:
                    to_send.append(g)
                case BrakeDecision.CANCEL:
                    self.action_log.record(g.plan, status="cancelled", reason="final_check")
                case BrakeDecision.DELAY:
                    self.sender.schedule(g, delay=self.config.send.min_interval_same_chat)
                    self.action_log.record(g.plan, status="scheduled", reason="brake_delay")
                case BrakeDecision.MERGE:
                    # MVP 降级: 不做真正的多 action 合并，仅重新生成当前 action
                    merged = self.content_gen.generate([g.plan])
                    if merged:
                        to_send.append(merged[0])
        self.sender.schedule_many(to_send)
        self.action_log.record_scheduled(to_send)

    def run_active_tick(self):
        """主动发言 tick。DecisionEngine 内的 LLM 调用失败也需 emit StateEvent。"""
        if self.state.mode != RuntimeMode.RUNNING:
            return
        
        self._drain_ingest()
        self._drain_state_events()
        self.state.tick(datetime.now())
        
        try:
            decision = self.engine.decide(TickType.ACTIVE)
        except LLMError as e:
            # Phase 3.2 LLM 意图判断失败
            self.state_events.put(StateEvent(
                event_id=uuid4(), event_type=StateEventType.LLM_FAILED,
                payload={"tick_type": "active"}, timestamp=datetime.now()
            ))
            return
        
        if not decision.plans:
            return
        self.action_log.record_planned(decision.plans)
        
        try:
            generated = self.content_gen.generate(decision.plans)
        except LLMError as e:
            self.state_events.put(StateEvent(
                event_id=uuid4(), event_type=StateEventType.LLM_FAILED,
                payload={"tick_type": "active", "chat_ids": [p.chat_id for p in decision.plans]},
                timestamp=datetime.now()
            ))
            self.action_log.record_batch(decision.plans, status="cancelled", reason="llm_failed")
            return

        self.action_log.record_generated(generated)
        to_send = []
        for g in generated:
            decision = self.brake.final_check(g)
            match decision:
                case BrakeDecision.ALLOW:
                    to_send.append(g)
                case BrakeDecision.CANCEL:
                    self.action_log.record(g.plan, status="cancelled", reason="final_check")
                case BrakeDecision.DELAY:
                    self.sender.schedule(g, delay=self.config.send.min_interval_same_chat)
                    self.action_log.record(g.plan, status="scheduled", reason="brake_delay")
                case BrakeDecision.MERGE:
                    # MVP 降级: 不做真正的多 action 合并，仅重新生成当前 action
                    merged = self.content_gen.generate([g.plan])
                    if merged:
                        to_send.append(merged[0])
        self.sender.schedule_many(to_send)
        self.action_log.record_scheduled(to_send)
    
    def run_concentrate_tick(self):
        """浓缩 tick。LLM 失败需 emit StateEvent。"""
        if self.state.mode != RuntimeMode.RUNNING:
            return
        
        for chat in self.memory.get_active_chats():
            if chat.should_concentrate():
                try:
                    ConcentrateJob.run(chat, self.memory, self.llm_client)
                except LLMError as e:
                    self.state_events.put(StateEvent(
                        event_id=uuid4(), event_type=StateEventType.LLM_FAILED,
                        payload={"tick_type": "concentrate", "chat_id": chat.chat_id},
                        timestamp=datetime.now()
                    ))
                    # 浓缩失败不得丢 read 消息，跳过本次，下一轮重试
```

## 9. 决策层设计

### 9.1 `DecisionEngine`（硬边界）

**DecisionEngine 允许的 LLM 调用：仅限主动发言意图判断。**

| 允许 | 禁止 |
|---|---|
| `ActiveSpeakGate.evaluate_with_llm()` → `DecisionOutput` | 生成回复文本 |
| `DecisionOutput` 只能包含 `should_speak / chat_id / intent / reason` | 调用 ContentGenerator |
| | 任何包含最终回复文本的输出 |

```python
class DecisionEngine:
    """
    纯决策引擎。
    唯一 LLM 调用: Phase 3.2 主动发言意图判断 → DecisionOutput (Pydantic schema)。
    """
    
    def decide(self, tick_type: TickType) -> DecisionResult:
        if tick_type == TickType.PASSIVE:
            return self._decide_passive()
        elif tick_type == TickType.ACTIVE:
            return self._decide_active()
    
    def _decide_passive(self) -> DecisionResult:
        # Phase 2: 纯规则，不调 LLM
        plans = self.passive_judge.evaluate(self.memory.get_active_chats())
        return DecisionResult(plans=plans, trigger_type="passive_tick")
    
    def _decide_active(self) -> DecisionResult:
        # Phase 3.1: 硬性门槛（纯规则）
        if not self.active_gate.check_hard_thresholds():
            return DecisionResult(plans=[], trigger_type="active_tick")
        
        # Phase 3.2: LLM 意图判断（唯一允许的 LLM 调用）
        # 输出: DecisionOutput {should_speak, chat_id, intent, reason}
        decision_output = self.active_gate.evaluate_with_llm(
            self.memory.get_active_chats(), self.llm, self.memory
        )
        if decision_output and decision_output.should_speak:
            # 显式转换: DecisionOutput → ActionPlan
            plan = ActionPlan(
                action_id=generate_action_id(),
                action_type=ActionType.ACTIVE_SPEAK,
                priority=Priority.P2,
                chat_id=decision_output.chat_id,
                chat_type="group",
                trigger_message=None,
                trigger_type="active_intent",
                reason=decision_output.reason or decision_output.intent,
            )
            plans = [plan]
        else:
            plans = []
        return DecisionResult(plans=plans, trigger_type="active_tick")
```

        P2 (active):   命中 min_interval → CANCEL

    def pre_send_check(self, action: GeneratedAction,
                       state: BotState, memory: MemoryManager) -> BrakeDecision:
        """发送前纯规则复查（见 §10.4）"""

## 11. 优雅停止（含 IncomingQueue 处理）

```python
class ShutdownManager:
    def shutdown(self, drain_pending: bool = False):
        """
        1. stop ncatbot: 停止接收新事件
        2. BotState.transition(STOPPING)
        3. MessageIngestor.drain_all(): 消费所有已入队消息，不丢失
        4. MainLoop.stop(): 设置 shutdown_event
        5. DelaySendScheduler.stop(drain=drain_pending)
           - drain=True: 发送已到期任务，取消未到期
           - drain=False: 全部取消写 action_log
        6. BotState.serialize() → state.json
        7. 关闭所有 SQLite 连接
        8. BotAdapter.stop()
        9. BotState.transition(STOPPED)
        """

class DelaySendScheduler:
    def stop(self, drain: bool = False):
        self._shutdown.set()
        if drain:
            while self._heap and self._heap[0].fire_at <= time.time():
                self._execute_send(heapq.heappop(self._heap))
            for item in self._heap:
                self.action_log.record(item.action, status="cancelled", reason="shutdown")
        else:
            for item in self._heap:
                self.action_log.record(item.action, status="cancelled", reason="shutdown")
```

## 13. 与旧版的关键差异总结

| 维度 | 旧版 | 新版 |
|---|---|---|
| 配置 | 单文件 `config.yaml` | `conf/` + Pydantic schema + MemoryThresholds 按类型 |
| 消息入口 | 直接写 MemoryManager | MessageIngestor: 解析→去重→写入（主循环线程） |
| 消息去重 | 无 | raw_message_log + UNIQUE(message_id) + 清理策略 |
| 记忆 | 两级（内存 + 单表覆盖） | 四级（即时→短期→pending→长期追加+merge+archive） |
| MemoryEntry | timestamp + content | + memory_type/confidence/source_message_ids/access_count/expires_at/is_active |
| 长期记忆过期 | 无 | expires_at → is_active=0 归档（不删除） |
| 决策输出 | LLM 一手包办 | DecisionEngine 纯决策，仅主动意图判断允许 LLM |
| 决策/生成分离 | 同一对象 | ActionPlan (意图) + GeneratedAction (内容) |
| 编排 | 无 | Orchestrator: 决策→刹车→生成→刹车→调度 |
| 主循环 | 单频率 | 三 tick 解耦 (被动 3s/主动 60s/浓缩 300s) |
| 被动 tick | 15s | 3s (纯规则) |
| 发送调度 | 每回复一线程 | DelaySendScheduler 单线程 + 优先队列 |
| 发送确认 | 无 | pre_send_check MVP 8 条纯规则 |
| 刹车决策 | bool | BrakeDecision: ALLOW/CANCEL/DELAY/MERGE |
| P0 vs min_interval | 直接拦截 | P0 ALLOW, P1 DELAY, P2 CANCEL |
| BotState 写入 | 多线程直接写 | 主循环独占 + StateEventQueue 单写者 |
| BotState tick | 次数驱动 | 时间驱动 (dt_seconds) |
| RuntimeMode | `is_character: bool` | 8 态 + 行为表 + Orchestrator 分支 |
| 发送结果 | `bool` | `SendResult` (6 种状态) |
| ActionLog | 无 | 幂等 (action_id UNIQUE) + 完整状态机 (planned→sending→sent) |
| DEGRADED 回复 | 系统报错感 | 角色化短句 + 低频 + 可沉默 |
| 优雅停止 | 无 | ShutdownManager + drain 策略 + IncomingQueue 处理 |
| 线程安全 | 基本无 | 明确的三线程模型 + lock 边界 + 单写者规则 |
